Detailed Description

Modality class that implements a texture-based modality, which uses information from a ColorCamera to calculate the gradient vector and Hessian matrix that are used by a Link and Optimizer to update the Body pose.

The modality required a FocusedSilhouetteRenderer that is used to reconstruct 3D feature points for keyframes. The modality is able to measure occlusions using images from a referenced DepthCamera that is close to the referenced ColorCamera and model occlusions using renderings from a FocusedDepthRenderer object.

Parameters
color_camera_ptrreferenced ColorCamera from which images are taken.
silhouette_renderer_ptrreferenced FocusedSilhouetteRenderer that is used to reconstruct 3D feature points for keyframes. It has to be configured with the IDType BODY.
depth_camera_ptrreferenced DepthCamera that is used to for measured occlusion handling.

focused_image_sizespecifies the size of the image crop in which features are detected.
descriptor_distance_thresholdspecifies the minimum difference of the Hamming distance between the best and second-best match.
tukey_norm_constantdefines the maximum expected value for valid residual errors in image space in pixel.
standard_deviationsuser-defined standard deviation for each iteration in pixels. If fewer values than n_corr_iterations are given, the last provided standard deviation is used.
max_keyframe_rotation_differencedefines after which rotational difference in radian a new keyframe is reconstructed.
max_keyframe_agespecifies after how many iterations a new keyframe has to be generated, even if the rotational difference criterion is not fulfilled.
n_keyframeshow many keyframes are considered at the same time.
orb_n_featuresnumber of features considered by ORB detector.
orb_scale_factorscale factor used by ORB detector.
orb_n_levelsnumber of levels considered by ORB detector.
brisk_thresholdthreshold used by BRISK detector.
brisk_octavenumber of detection octaves considered by BRISK detector.
brisk_pattern_scalescale applied to the pattern that is used to sample the neighborhood of a BRISK keypoint.
daisy_radiusradius of the DAISY descriptor at initial scale.
daisy_q_radiusradial range division quantity for DAISY descriptor.
daisy_q_thetaangular range division quantity for DAISY descriptor.
daisy_q_histgradient orientations range division quantity for DAISY descriptor.
freak_orientation_normalizedenable orientation normalization for FREAK descriptor.
freak_scale_normalizedenable scale normalization for FREAK descriptor.
freak_pattern_scalescaling of the FREAK description pattern.
freak_n_octavesnumber of octaves covered by detected keypoints for FREAK descriptor.
sift_n_featuresnumber of best features to retain by SIFT detector.
sift_n_octave_layersnumber of layers in each octave considered by SIFT detector
sift_contrast_thresholdcontrast threshold used to filter out weak features in semi-uniform regions by SIFT detector.
sift_edge_thresholdthreshold used to filter out edge-like features by SIFT detector.
sift_sigmasigma of the Gaussian applied to the input image at the octave #0 by SIFT detector.
measured_occlusion_radiusradius in meter that defines the area in which depth measurements from a DepthCamera are considered for occlusion handling.
measured_occlusion_thresholddefines how much smaller the minimum value from the DepthCamera is allowed to be compared to the value that is expected from the current estimate.
modeled_occlusion_radiusradius in meter that defines the area in which depth measurements from a FocusedDepthRenderer are considered for occlusion handling.
modeled_occlusion_thresholddefines how much smaller the minimum value from the FocusedDepthRenderer is allowed to be compared to the value that is expected from the current estimate.

visualize_points_resultvisualize keyframe points during VisualizeResults().
visualize_points_depth_image_resultvisualize keyframe points on the depth image, which is used for occlusion handling, during VisualizeResults().

visualize_points_depth_rendering_resultvisualize keyframe points on the depth rendering, which is used for occlusion handling, during VisualizeResults().
