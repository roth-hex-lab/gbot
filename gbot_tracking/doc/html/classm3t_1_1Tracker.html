Detailed Description

Class that coordinates Refiner, Publisher, Subscriber, Detector, Optimizer, Modality, Renderer, Viewer, and Camera objects to track multiple Body objects.

Using the method RunTrackerProcess(), the tracker process is started. With the parameters execute_detection and start_tracking, the function allows to specify if detection and tracking should be started automatically. If they are not started automatically, the user can press the D key for the detection and the T key to start tracking. With X, both the detection is executed and tracking is started. With S tracking stops while Q quits tracking. In addition to pressing keys on the keyboard, the user can call QuitTrackerProcess(), ExecuteDetection(), StartTracking(), and StopTracking() from another thread to control the tracker process. All methods allow to specify names of optimizers that should be considered by the respective command. If no name is specified, all optimizers are considered. During SetUp() all required objects like Link, Constraint, Modality, Model, Renderer, RendererGeometry, ColorHistograms Camera, and Body objects are derived from objects that are provided by the user. SetUp() calls the SetUp() method of all referenced objects in the correct order.

n_corr_iterationsnumber of times new correspondences are established.
n_update_iterationsnumber of times the pose is updated for each correspondence iteration.
synchronize_cameraswhen true, Camera objects wait for new images to arrive. Otherwise they do not wait. Instead, the tracker waits until the cycle_duration has passed and starts a new cycle.
start_tracking_after_detectionwhen true bodies are directly tracked after a successful detection.
cycle_durationduration of a cycle if the flag synchronize_cameras is set to false.
visualization_timetime in milliseconds visualizations from Modality objects are shown. 0 corresponds to infinite time.
viewer_timetime in milliseconds images from Viewer objects are shown. 0 corresponds to infinite time.
